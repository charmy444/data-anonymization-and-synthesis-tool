# SDA (Synthetic Data Anonymizer)

Простой учебный проект для кейса: генерация синтетических CSV и анонимизация CSV без использования реальных персональных данных.

## Что это за проект

`SDA` = `Synthetic Data Anonymizer`  
Инструмент для **синтетических данных** и **анонимизации**.

Цель MVP:
1. Сгенерировать тестовые CSV (например, `users.csv`, `orders.csv`).
2. Анонимизировать загруженный CSV по правилам колонок:
   - masking (маскирование),
   - redaction (удаление/скрытие),
   - pseudonymization (стабильная подмена).

## Минимальный целевой пользовательский сценарий (MVP)

1. Пользователь запускает `generate` и получает `users.csv`/`orders.csv`.
2. Пользователь запускает `anonymize` для своего CSV и выбирает правила.
3. Пользователь получает безопасный CSV для тестов/учебы.

## Что важно для защиты кейса

1. Репозиторий должен запускаться по понятной инструкции.
2. Должны быть реализованы 2 шаблона генерации и 3 метода анонимизации.
3. Ошибки должны быть понятны человеку (а не «traceback без объяснения»).
4. README должен содержать примеры команд и примеры CSV.
